# 📘 **Chapter 2.3: Activation Functions**

This chapter explains one of the most critical components in neural networks:

👉 **Activation Functions — the reason neural networks can learn complex patterns**

---

## 🔹 1. Why Do We Need Activation Functions?

Without activation, a neuron does only this:

[
y = w_1x_1 + w_2x_2 + b
]

👉 This is just a **straight line (linear equation)**

---

### Problem

Even with many layers:

* Linear + Linear + Linear = Linear

👉 The network becomes too simple

---

### Solution

👉 Add a **non-linear function**

This is called the **activation function**

---

### Simple Analogy

Think of:

* Raw score → marks calculation
* Activation → pass/fail decision

Without decision step → no real intelligence

---

## 🔹 2. What is an Activation Function?

### Definition

👉 A function applied after neuron computation that decides:

* Should the neuron activate?
* How strong should the output be?

---

### General Form

y = f(z)

Where:

* (z = Wx + b)
* (f) = activation function

---

## 🔹 3. Types of Activation Functions

---

## 🔸 3.1 Sigmoid Function

![Image](https://images.openai.com/static-rsc-4/XJ0Gc2AmD7AVIS1QJC7r_uktdmpj5PXY67PtxeDMCHB2eZPyHbevAXiSCdE-JKMg9jFM7DSUD3hgApYOnis7lNfC5kepr0vWa6XO0xKVI78hZJ9ClIt91xvtva3btV3Bpg_LTTYz15U3pAYy0bqLa3rPd4IOl7Pb9teflBx8vanoK0itrJok9immFAa__iEp?purpose=fullsize)

### Formula

[
\sigma(z) = \frac{1}{1 + e^{-z}}
]

---

### Output Range

👉 Between **0 and 1**

---

### Use Case

* Probability output
* Binary classification

---

### Intuition

* Very small input → near 0
* Very large input → near 1

👉 Like a **yes/no probability**

---

### Problem

* Vanishing gradient (learning slows down)

---

## 🔸 3.2 Tanh Function

![Image](https://images.openai.com/static-rsc-4/VknS8wmNJzhg7PXapLyg7jNhvsb1CnT5c62A9tJ12Actr5RvPpY35EsjOPrEyuReLCtY3MnXK7N5TZwmv1uazUZQCpb-XFUIKkZtJM7h4Y50Por4xrrQGOPe1-dF7M0SqzXpW91P9xoKaYAKbFHBt8N3K00hxk_Yk44iZoPj7pL3UD9_A8P0pozOKJx4SaAY?purpose=fullsize)

### Output Range

👉 Between **-1 and 1**

---

### Why Better than Sigmoid?

* Centered around zero
* Faster learning

---

### Intuition

* Negative input → negative output
* Positive input → positive output

---

## 🔸 3.3 ReLU (Rectified Linear Unit)

![Image](https://images.openai.com/static-rsc-4/Ay49LJQz7N1wz5k_fYu11LtQvVhMPW9ity1wPsEBzy4MCSjp6IacVa9Z4VvY2utVkpGeOImXz2wqPyk6e9iY1Yu4pF-lv9lEWgLc_63Mj03qVjs7Bp56DBrkhcda0Vr37ww2zb4EYNusA-DwjYu43ifQjfqrWQPioeX5FBO-R_xVhcqFkfqu8S07kE7TAYk0?purpose=fullsize)

### Formula

[
f(z) = \max(0, z)
]

---

### Behavior

* Negative → 0
* Positive → same value

---

### Why Popular?

* Simple
* Fast
* Works well in deep networks

---

### Intuition

👉 Like a switch:

* Negative signal → OFF
* Positive signal → ON

---

## 🔸 3.4 Leaky ReLU

### Idea

Fixes problem of ReLU (dead neurons)

---

### Behavior

* Small slope for negative values
* Normal for positive

---

### Advantage

👉 Prevents neurons from “dying”

---

## 🔸 3.5 Softmax Function

### Use Case

👉 Multi-class classification

---

### Example

Output:

```id="y7l0xa"
[cat: 0.7, dog: 0.2, bird: 0.1]
```

👉 Probabilities sum to 1

---

## 🔹 4. Comparison of Activation Functions

| Function   | Range                  | Use                   | Problem              |
| ---------- | ---------------------- | --------------------- | -------------------- |
| Sigmoid    | 0 to 1                 | Binary classification | Vanishing gradient   |
| Tanh       | -1 to 1                | Hidden layers (older) | Still gradient issue |
| ReLU       | 0 to ∞                 | Most common           | Dead neurons         |
| Leaky ReLU | small negative allowed | Improved ReLU         | Slight complexity    |
| Softmax    | Probabilities          | Multi-class output    | Computational cost   |

---

## 🔹 5. Why ReLU is Most Used Today

👉 Because it:

* Trains faster
* Avoids saturation
* Works well with deep networks

---

### Simple Intuition

* Keeps useful signals
* Removes noise (negative values)

---

## 🔹 6. Choosing the Right Activation

### Hidden Layers

👉 ReLU (default choice)

---

### Output Layer

| Problem Type | Activation    |
| ------------ | ------------- |
| Binary       | Sigmoid       |
| Multi-class  | Softmax       |
| Regression   | None / Linear |

---

## 🔹 7. Key Insight

👉 Activation functions are what make neural networks:

* Powerful
* Flexible
* Capable of learning complex patterns

---

Without them:
👉 Neural networks = simple linear models

---

## 🔹 8. Common Mistakes

### ❌ Using sigmoid everywhere

✔️ Use ReLU for hidden layers

---

### ❌ Ignoring output activation

✔️ Match activation with problem type

---

## 🎯 Summary (In One Line)

👉 Activation functions introduce **non-linearity**, allowing neural networks to learn complex real-world patterns.
