# 📘 **Chapter 2.1: Linear Algebra Basics for Neural Networks**

This chapter gives you the **minimum linear algebra needed to understand ANN**.

👉 Don’t worry—we will keep it simple and practical.

---

## 🔹 1. Why Linear Algebra in ANN?

Neural networks deal with:

* Many inputs
* Many neurons
* Many connections

👉 Handling all this efficiently requires **vectors and matrices**

---

### Simple Analogy

Think of:

* A single number → one student
* A list of numbers → a class
* A matrix → entire school data

👉 Linear algebra helps manage this data easily.

---

## 🔹 3. Why Vectors Matter in ANN

👉 Instead of processing one input at a time:

* ANN processes **multiple inputs together**

---

### Example

Inputs:

```
x = [x₁, x₂, x₃]
```

Weights:

```
w = [w₁, w₂, w₃]
```

👉 We combine them using vector operations

---

## 🔹 4. Dot Product (Most Important Operation)

This is the **core computation in neural networks**

w \cdot x = \sum_{i=1}^{n} w_i x_i

---

### Simple Explanation

Multiply corresponding values and add:

```
x = [2, 3]
w = [4, 5]

Result = (2×4) + (3×5) = 8 + 15 = 23
```

---

### Real-Life Analogy

Think of:

* Inputs = marks
* Weights = importance

👉 Dot product = **weighted score**

---

## 🔹 5. Matrix Multiplication

Neural networks use this to process **many neurons at once**

---

### Idea

Instead of:

* One neuron → one calculation

We do:

* Many neurons → one matrix operation

---

### Example

Input:

```
[ x₁, x₂ ]
```

Weights:

```
[ w₁₁  w₁₂
  w₂₁  w₂₂ ]
```

👉 Output:

```
[ y₁, y₂ ]
```

---

### Intuition

Matrix multiplication =
👉 “Apply many neurons simultaneously”

---

## 🔹 6. Transpose of a Matrix

### Definition

Flipping rows into columns

---

### Example

```
A =
[1 2
 3 4]

Aᵀ =
[1 3
 2 4]
```

---

### Why Useful?

* Align dimensions for multiplication
* Used in backpropagation

---

## 🔹 7. Shape (Dimensions)

### Definition

Shape tells size of matrix

---

### Example

```
(3 × 2) → 3 rows, 2 columns
```

---

### Why Important?

👉 Matrix multiplication only works when:

```
(columns of A) = (rows of B)
```

---

### Simple Rule

If shapes don’t match → calculation fails

---

## 🔹 8. Bias as a Vector

Earlier we saw bias as a single number.

👉 In ANN:

* Each neuron has its own bias

So bias becomes:

```
b = [b₁, b₂, b₃]
```

---

## 🔹 9. Putting It All Together (Neuron in Vector Form)

Instead of computing one by one:

y = f(Wx + b)

---

### Meaning

* (W) → weight matrix
* (x) → input vector
* (b) → bias vector
* (f) → activation function

---

👉 This is the **core equation of neural networks**

---

## 🔹 10. Why This Matters in Practice

In Python (NumPy / PyTorch / TensorFlow):

* Everything is vectorized
* No loops needed

👉 Faster and more efficient

---

### Example (Conceptual)

Instead of:

```
for each neuron:
    compute output
```

We do:

```
output = W @ x + b
```

👉 Neural networks use vectors and matrices to efficiently compute **weighted sums of inputs**.
