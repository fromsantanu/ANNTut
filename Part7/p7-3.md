# 📘 **Chapter 7.3: Hyperparameter Tuning**

This chapter focuses on one of the most practical skills in deep learning:

👉 **Choosing the right settings (hyperparameters) to improve model performance**

---

## 🔹 1. What are Hyperparameters?

### Definition

👉 **Hyperparameters** are values that:

* Are set **before training starts**
* Control how the model learns

---

### Examples

* Learning rate
* Batch size
* Number of layers
* Number of neurons

---

### Important Difference

| Type            | Meaning                               |
| --------------- | ------------------------------------- |
| Parameters      | Learned automatically (weights, bias) |
| Hyperparameters | Set manually before training          |

---

👉 Hyperparameters = **control knobs**

---

## 🔹 2. Why Hyperparameter Tuning Matters

👉 Same model + different hyperparameters = very different results

---

### Example

* High learning rate → unstable
* Low learning rate → slow

---

👉 Tuning helps find the **best combination**

---

## 🔹 3. Key Hyperparameters in Neural Networks

---

### 🔸 1. Learning Rate

👉 Controls step size during training

* Too high → overshoot
* Too low → slow learning

---

### 🔸 2. Batch Size

👉 Number of samples per update

* Small → noisy but faster
* Large → stable but slow

---

### 🔸 3. Number of Epochs

👉 How many times data is used

* Too few → underfitting
* Too many → overfitting

---

### 🔸 4. Number of Layers

👉 Controls model complexity

---

### 🔸 5. Number of Neurons

👉 Determines model capacity

---

### 🔸 6. Activation Function

👉 Affects learning behavior

---

### 🔸 7. Optimizer

👉 Controls how weights are updated

---

## 🔹 4. How to Tune Hyperparameters

---

## 🔸 4.1 Manual Tuning

👉 Try different values manually

---

### Advantage

* Simple

---

### Disadvantage

* Time-consuming

---

## 🔸 4.2 Grid Search

👉 Try all combinations

---

### Example

```id="1ahm9s"
learning_rate = [0.01, 0.001]
batch_size = [32, 64]
```

---

👉 Test all combinations

---

### Problem

* Expensive for large search space

---

## 🔸 4.3 Random Search

👉 Randomly select combinations

---

### Advantage

* Faster than grid search
* Often effective

---

## 🔸 4.4 Bayesian Optimization (Advanced)

👉 Uses past results to choose better parameters

---

### Advantage

* Efficient
* Smart search

---

## 🔹 6. Practical Strategy

---

### Step-by-Step

1. Start with default values
2. Tune learning rate first
3. Adjust batch size
4. Modify architecture
5. Apply regularization

---

👉 Tune one thing at a time

---

## 🔹 7. Validation Set

👉 Use separate data for tuning

---

### Data Split

* Training → learn
* Validation → tune
* Testing → final evaluation

---

👉 Avoids overfitting during tuning

---

## 🔹 8. Common Patterns

---

### Learning Rate

* Try: 0.1 → 0.01 → 0.001

---

### Batch Size

* Common: 16, 32, 64

---

### Epochs

* Use early stopping

---

## 🔹 9. Common Mistakes

---

### ❌ Changing many parameters at once

✔️ Change one at a time

---

### ❌ Ignoring validation set

✔️ Always validate

---

### ❌ Over-tuning

✔️ May overfit validation data

---

## 🔹 10. Real-Life Analogy

### Cooking Recipe

* Adjust salt, spice, cooking time

👉 Small changes → big difference

---

## 🔹 11. Key Insight

👉 Hyperparameter tuning is:

* Not exact science
* Requires experimentation

---

👉 Experience improves results

---

## 🎯 Summary (In One Line)

👉 Hyperparameter tuning is the process of selecting the best settings for a neural network to achieve optimal performance.
