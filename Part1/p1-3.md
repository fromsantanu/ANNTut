# 📘 **Chapter 1.3: Basic Terminology in Artificial Neural Networks**

This chapter builds the **language of neural networks**.
Once students understand these terms, everything else becomes much easier.

---

## 🔹 1. Why Terminology Matters

Think of learning cricket:

* You must know terms like *batting, bowling, fielding*

👉 Similarly, ANN has its own basic terms:

* Input
* Weight
* Bias
* Activation
* Loss

---

## 🔹 2. Input (Features)

### Definition

Inputs are the **data given to the network**

---

### Examples

| Problem            | Inputs                |
| ------------------ | --------------------- |
| House price        | size, location, rooms |
| Student result     | marks, attendance     |
| Disease prediction | age, symptoms         |

---

👉 Think of inputs as **ingredients in cooking**

---

## 🔹 3. Weights (Importance)

### Definition

Weights represent **how important each input is**

---

### Simple Example

* Maths marks may be more important than sports marks
* So maths gets a higher weight

---

👉 Like volume control:

* High weight → strong influence
* Low weight → weak influence

---

## 🔹 4. Bias (Adjustment Factor)

### Definition

Bias is an extra value added to adjust the output

---

### Real-Life Example

Even if a student scores low:

* Teacher may give **grace marks**

👉 That “extra push” = bias

---

## 🔹 5. Activation Function (Decision Maker)

### Definition

It decides **whether a neuron should activate (fire) or not**

---

### Why Needed?

Without activation:

* Model becomes too simple (linear only)

---

### Common Activations



| Function | Behavior                | Use           |
| -------- | ----------------------- | ------------- |
| ReLU     | Removes negative values | Most common   |
| Sigmoid  | Output between 0 and 1  | Probability   |
| Tanh     | Output between -1 and 1 | Balanced data |

---

👉 Think of activation as a **yes/no decision step**

---

## 🔹 6. Output

### Definition

The final result produced by the network

---

### Examples

| Problem              | Output          |
| -------------------- | --------------- |
| Spam detection       | Spam / Not Spam |
| Price prediction     | ₹ value         |
| Image classification | Cat / Dog       |

---

👉 Output = **final answer**

---

## 🔹 7. Loss Function (Error Measurement)

### Definition

Loss function measures **how wrong the prediction is**

---

### Example

* Actual price = ₹100
* Predicted price = ₹80
* Error = ₹20

👉 Loss tells the model:
**“You are this much wrong”**

---

### Common Types

* Mean Squared Error (MSE) → for regression
* Cross-Entropy → for classification

---

## 🔹 8. Gradient Descent (Learning Method)

### Definition

A method to **reduce error by adjusting weights**

---

### Simple Analogy

Imagine:

* You are going downhill in fog
* You take small steps downward

👉 Eventually, you reach the lowest point

That is gradient descent.

---

## 🔹 9. Learning Rate (Step Size)

### Definition

Controls how big steps we take while learning

---

### Analogy

* Too big steps → may miss the target
* Too small steps → very slow learning

👉 Need a **balanced step size**

---

## 🔹 10. Epoch

### Definition

One complete pass through the entire dataset

---

### Example

If dataset has 1000 samples:

* 1 epoch = model sees all 1000 once

---

👉 Like revising a full syllabus once

---

## 🔹 11. Batch and Iteration

### Batch

* Small subset of data

### Iteration

* One update step using one batch

---

### Example

Dataset = 1000 samples
Batch size = 100

* 1 epoch = 10 iterations

---

👉 Like studying chapter by chapter instead of whole book at once

---

## 🔹 12. Model

### Definition

The complete neural network with:

* Structure
* Weights
* Parameters

---

👉 Model = “trained system ready to predict”

---

## 🔹 13. Parameters vs Hyperparameters

### Parameters (Learned automatically)

* Weights
* Bias

---

### Hyperparameters (Set by us)

* Learning rate
* Number of layers
* Batch size

---

👉 Simple difference:

* Parameters → learned
* Hyperparameters → decided

---

## 🔹 14. Putting It All Together

### Flow of ANN

1. Input enters
2. Multiply by weights
3. Add bias
4. Apply activation
5. Generate output
6. Compare with actual (loss)
7. Adjust weights (learning)

---

## 🎯 Summary (Quick View)

| Term       | Meaning        |
| ---------- | -------------- |
| Input      | Data           |
| Weight     | Importance     |
| Bias       | Adjustment     |
| Activation | Decision       |
| Output     | Result         |
| Loss       | Error          |
| Epoch      | Full data pass |
